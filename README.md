# Dinesh Kumar M B - Developer Portfolio

A modern, highly interactive, and SEO-optimized developer portfolio built with React, Vite, Tailwind CSS, and Framer Motion. 

This portfolio showcases my expertise as a Full-Stack MERN & AI Developer, featuring detailed project case studies, dynamic GitHub activity statistics, and an engaging UI with dark/light mode support.

## 🚀 Live Preview
**[View Portfolio Here](https://portfolio-five-chi-11.vercel.app/)**

## ✨ Features

- **Modern Tech Stack:** Built with React 18, Vite for blazing fast builds, and Tailwind CSS for utility-first styling.
- **Animations & Interactivity:** Fluid page transitions and micro-interactions powered by Framer Motion.
- **SEO Optimized:** Fully configured with `react-helmet-async`, Open Graph tags, Twitter cards, JSON-LD Schema, and a dynamically mapped `sitemap.xml`.
- **Responsive Design:** Flawless viewing experience across mobile, tablet, and desktop devices.
- **Dark/Light Mode:** First-class support for theme switching with custom glassmorphism components.
- **Production Ready:** Fully configured for Vercel deployment with proper SPA fallback rewrites via `vercel.json`.

## 🛠️ Tech Stack

- **Framework:** React.js + Vite
- **Styling:** Tailwind CSS
- **Routing:** React Router v6
- **Animations:** Framer Motion
- **SEO:** React Helmet Async
- **Deployment:** Vercel

## ⚙️ Getting Started

To run this project locally, follow these steps:

1. **Clone the repository:**
   ```bash
   git clone https://github.com/dineshkumar-mb/portfolio.git
   cd portfolio
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Start the development server:**
   ```bash
   npm run dev
   ```
   Open `http://localhost:5173` in your browser.

4. **Build for production:**
   ```bash
   npm run build
   ```

## 📂 Key Architecture

- `src/Components/` - Houses reusable UI components (`Header.jsx`, `Project.jsx`, `GithubSection.jsx`, `SEO.jsx`, etc.)
- `src/Pages/` - Route-level views (e.g., `ProjectCaseStudy.jsx`)
- `public/` - Static assets including custom `favicon.svg`, `sitemap.xml`, `robots.txt`, and `_redirects`.
- `vercel.json` - Custom Vercel routing rules to prevent 404 errors on dynamic routes.
